# Изменения портала — сводка

Ниже — что именно поправлено по вашему списку. Порядок запуска не изменился:

```
npm install
npm run init-db
npm run build
npm start
# для разработки:
npm run dev:server
npm run dev:client
```

> Добавлена одна зависимость бэкенда — `docx` (генерация Word). `npm install` подтянет её автоматически.

---

## 1. Редактирование уже добавленных событий календаря
- В карточке события теперь две кнопки: ✎ (изменить) и × (удалить) — у автора события и у admin/manager.
- Модалка события работает в двух режимах: «Новое событие» и «Изменить событие» (тип, название, дата, время, «весь день», описание).
- Бэкенд: `PATCH /api/portal/events/:id` расширен — теперь принимает не только `starts_at/title/all_day`, но и `descr` и `kind`. В выдаче событий добавлен флаг `can_edit`.
- Файлы: `client/src/portal/Calendar.jsx`, `client/src/portal/portal.css`, `routes/portalLife.js`.

## 2. Глобальный поиск (Ctrl+K): без удалённого + пословный префиксный поиск
- **Удалённое больше не всплывает.** При удалении документа/новости/заявки/задачи/события запись сразу убирается из поискового индекса (`removeFromIndex`), а не ждёт переиндексации (раз в 5 минут).
- **Побуквенный поиск.** Ctrl+K теперь ищет литерально, по началу слова: `a` → всё, что начинается с `a`; `ayat` → только «ayat…» (не попадают связи с `a`, `ay`, `aya`). Каждое слово запроса должно найтись как начало слова в заголовке или тексте.
- Семантический RAG «Спроси у ДиДи» остался семантическим — менялся только глобальный поиск.
- Файлы: `lib/rag.js` (новые `prefixSearch`, `removeFromIndex`), `routes/assistant.js`, `routes/portalDocs.js`, `routes/portalLife.js`, `routes/portalWork.js`, `client/src/portal/PortalApp.jsx`.

## 3. Заголовок PDF по центру + экспорт в DOCX (Word)
- Заголовок документа в PDF выравнивается по центру (проверено: центр строки совпадает с центром страницы).
- **Добавлен DOCX.** Рядом с «⬇ PDF» появилась кнопка «⬇ Word» — скачивается редактируемый `.docx` на том же фирменном бланке ЦЦР (логотип, двуязычная шапка, реквизиты, **заголовок по центру**, нумерованные разделы жирным, блок «УТВЕРЖДЕНО», подпись, колонтитул со стр.). Открывается и правится в Word.
- Бэкенд: новый модуль `docDocx.js` и эндпоинт `GET /api/portal/docs/:id/docx`.
- Файлы: `docDocx.js` (новый), `routes/portalDocs.js`, `client/src/portal/Documents.jsx`, `client/src/portal/portal.css`.

## 4. Разделение задач между OpenAI и Gemini
Введены явные роли в `lib/ai.js`:
- **OpenAI (ChatGPT) — «мозг» интерактива** (`aiBrain`/`aiText`, OpenAI приоритетно): команды ассистента, голос, ответы RAG, генерация документов, тексты для человека.
- **Gemini — «аналитик» фоновых пачек** (`aiBatch`, Gemini приоритетно, OpenAI — фолбэк): мультиязычный дайджест новостей и разбор заявок (большой контекст/квота). Раньше эти задачи звали Gemini напрямую без фолбэка — теперь оба провайдера загружены и подстраховывают друг друга.
- Файлы: `lib/ai.js`, `lib/feed.js`, `routes/adminOps.js`.

## 5. Текстовый/голосовой помощник — приоритет ChatGPT
- В голосовой ветке ассистент раньше сначала прогонял фразу через простой локальный парсер и только потом — через ИИ. Теперь **приоритет у разбора OpenAI** (понимает свободные формулировки, перенос встреч, несколько действий за фразу), локальный парсер — только страховка, если ИИ действий не вернул.
- Текстовые команды и так шли через OpenAI (сервер `/api/assistant/command`) — оставлено, локальный парсер срабатывает лишь при недоступности сети.
- Файлы: `client/src/portal/VoiceAgent.jsx` (+ роли в `lib/ai.js`).

## 6. QR-код визитки (vCard) — читается корректно
- Причина «битых символов, кроме имени»: vCard экранировался слишком агрессивно (`;` и `,` экранировались во всех полях подряд), а имя в структурном поле `N` не имело нормальной структуры — многие сканеры показывали мусор.
- Исправлено: корректная структура `N:Фамилия;Имя;;;`, экранирование только там, где требует спецификация, поле `ORG` собирается как «Организация;Подразделение» со структурным разделителем. Уровень коррекции ошибок повышен до `Q` (кириллица в UTF-8 «тяжёлая»).
- Проверено сканером (zbar — движок телефонных камер): вся карточка (имя, организация, должность, телефон, навыки) считывается верно.
- Файл: `client/src/portal/PortalApp.jsx`.

## 7. Кнопка помощника перекрывала «Отправить» в чатах
- В открытом чате (личном/командном) плавающая кнопка ДиДи поднимается над строкой ввода и больше не накрывает кнопку «Отправить». Когда панель ассистента открыта — она остаётся на месте.
- Файл: `client/src/portal/portal.css`.

## 8. Готовая сборка
- Клиент собирается без ошибок (`npm run build`), все бэкенд-модули загружаются. Зип — исходники без `node_modules` и без билд-артефактов: соберите как обычно.
